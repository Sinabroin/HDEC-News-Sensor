# D7-AD-S 현장 기상(날씨) 소스 연동 계획 (조사·계획 전용)

작성 시점: D7-AD-R(단일 임원 대시보드 IA 정리) 작업 중. 이 문서는 **조사/계획 전용**이며,
D7-AD-R에서는 실제 기상 소스를 **연동하지 않는다**. 가짜 날씨 값을 만들지 않는다(rules.md §1).

## 배경 — D7-AD-R에서 바뀐 것

- 기존: 기상·날씨가 임원 브리핑 아코디언의 한 섹션("기상·날씨 기사 0건")으로 노출 →
  뉴스 수집 섹션처럼 보였고 항상 0건이라 방치된 인상.
- D7-AD-R: 기상·날씨는 **뉴스 수집 섹션이 아니다**.
  - 브리핑 아코디언의 `weather` 섹션은 계약(이메일/Teams 본문 파리티, `verify_dashboard_accordion_sections`)
    유지를 위해 그대로 두되, 헤더 카운트를 `기사 0건` 대신 **`데이터 소스 미연동`**으로 표기한다
    (`scripts/build_static_dashboard.py:_render_accordion_section`).
  - 대시보드 화면에는 **시장 탭에 `현장 기상` 카드**(`#siteWeatherCard`)를 지도/지역 placeholder로
    둔다. 실제 데이터 소스가 없으므로 지역 칩은 전부 `미연동`으로 표기하고 값/아이콘을 생성하지 않는다.

## 원칙 (변경 없음)

- **no fake weather.** 공개 무료 실측 기상 소스가 연동되기 전에는 온도/강수/풍속 등 어떤 값도 만들지 않는다.
- 연동 시에도 현장 좌표는 비공개(운영자 빌드 전용). 공개 빌드는 지역 집계(수도권/영남권/…)만 노출하고
  개별 현장 좌표·이름은 노출하지 않는다(site_watchlist public-safe 정책과 동일).

## 연동 후보 (제안만 — 이번에 구현하지 않음)

각 후보는 "무료/키 요건", "국내 현장 커버리지", "정직한 표기 방법"을 먼저 검증한 뒤에만 착수한다.
실측 GET 프로빙으로 소스가 실제 응답하는지 확인하기 전에는 파서를 쓰지 않는다.

1. **기상청 단기예보/실황 (KMA)** — 국내 현장 1순위. 공공데이터포털 키 필요(무료).
   - 격자(nx, ny) 기반 → 현장 좌표를 격자로 변환. 좌표는 비공개(운영자 빌드 전용).
   - `data_mode` 후보: 실황=`delayed_market` 성격(관측 지연), 예보=예보 명시 라벨.
2. **Open-Meteo** — 키 불요(무료), 전 세계 좌표. 해외 현장/지사·법인 커버리지에 유리.
   - 국내 정확도는 KMA 대비 낮을 수 있어 국내는 KMA 우선, 해외는 Open-Meteo 검토.
3. **해양/태풍 특보** — 항만·해외 플랜트 현장 리스크 연계(별도 후보, 우선순위 낮음).

## 표기 계약 (연동 시)

- 지역 카드/지도 오버레이는 관측 시각과 소스를 함께 표기한다("현재값 주장" 금지 — 관측/예보 명시).
- 소스가 응답하지 않거나 키가 없으면 해당 지역은 `미연동`으로 되돌리고 마지막값을 위조하지 않는다.
- 특보(호우·강풍·폭염 등)는 현장 안전 액션과 연계하되 "위험 확정"이 아니라 "관찰"로 표기한다
  (site_watch_tree의 `!`=관찰 마커와 동일 톤).

## 착수 전 체크리스트

- [ ] KMA/Open-Meteo 실측 프로빙(무료/키/rate limit 확인)
- [ ] 현장 좌표 비공개 주입 경로(`SITE_WATCHLIST_PATH`류) 설계 — 공개 빌드 좌표 미노출 보장
- [ ] `현장 기상` 카드 렌더를 placeholder→실데이터로 교체하는 UI 계약(관측 시각/소스/미연동 폴백)
- [ ] 새 verifier: 좌표/이름 public 비노출 + 미연동 폴백 + 가짜 값 부재

## D7-AD-V 조사 보강 (엔드포인트·필드 확인 · 실연동은 D7-AD-W로 분리)

D7-AD-V에서 UI는 **명일(D+1) 정오 12:00 시공 리스크** 계약(`#siteWeatherCard`)을 유지하고 실제 기상
API는 붙이지 않는다. 아래는 조사로 **엔드포인트/필드 존재를 확인**한 결과이며, 실연동(파서·좌표 주입·
캐시·미연동 폴백)은 별도 커밋 **D7-AD-W**(또는 D7-AD-V2)에서 어댑터 단위로 진행한다. 가짜 날씨 값 없음.

### 카드 필드 ↔ 소스 매핑 (명일 정오 12:00 기준)

| 카드 항목 | Open-Meteo (해외+국내, 키불요) | 기상청 KMA (국내, 키필요) |
|-----------|-------------------------------|---------------------------|
| 강수확률 | `precipitation_probability` (hourly) | 단기예보 `POP` |
| 예상 강수량 | `precipitation` (hourly) | 단기예보 `PCP` |
| 풍속 | `wind_speed_10m` | 단기예보 `WSD` |
| 돌풍 | `wind_gusts_10m` ✅ | (단기예보에 돌풍 필드 없음 — Open-Meteo 보완) |
| 폭염·한파 | `temperature_2m` / `apparent_temperature` | `TMP`/`TMN`/`TMX` |
| 특보 | (없음 — 정부 특보 아님) | **기상특보 조회서비스**(호우/대설/강풍/폭염/한파 등) |
| 작업 리스크 등급 | 위 필드 파생(내부 규칙) | 위 필드 파생(내부 규칙) |

### 확인된 엔드포인트 (조사 시점 verified)

1. **Open-Meteo** — `https://api.open-meteo.com/v1/forecast` · **키 불요**(비상업 CC BY 4.0) · 위경도 직접 ·
   `forecast_days`로 D+1 12:00 커버(`timezone=Asia/Seoul`로 시각 정렬). hourly에 강수확률/강수량/풍속/
   **돌풍**/체감온도까지 있어 국내+해외 현장을 단일 API로 커버(모델에 KMA·JMA 포함). 한계: 정부 **특보**
   제품은 없음.
2. **기상청 단기예보(VilageFcst)** — `apis.data.go.kr/1360000/VilageFcstInfoService_2.0/getVilageFcst` ·
   **data.go.kr 무료 키 필요** · 격자(nx, ny) 변환 필요(현장 좌표 비공개). D+2까지 시간대별 → D+1 정오 포함.
   필드: `POP`(강수확률)/`PCP`(강수량)/`WSD`(풍속)/`TMP`·`TMN`·`TMX`(기온). **돌풍 필드 없음**.
3. **기상청 기상특보 조회서비스** — `apis.data.go.kr/1360000/WthrWrnInfoService/getWthrWrnList` ·
   **무료 키 필요** · 호우/대설/강풍/폭염/한파/태풍/풍랑/건조 등 주의보/경보. 카드 '특보' 열의 정부 소스.

### 권장 구성 (D7-AD-W)

- **국내 현장:** KMA 단기예보(POP/PCP/WSD/기온) + 기상특보(특보). 돌풍은 Open-Meteo `wind_gusts_10m`로 보완.
- **해외 현장/지사/법인:** Open-Meteo 단일(키불요·글로벌). 특보는 해당국 소스 부재 시 `미연동` 유지.
- **좌표 비공개:** nx/ny·위경도는 운영자(비공개) 빌드 전용 주입, 공개 빌드는 지역 집계만(현 정책 동일).
- **정직 표기:** 예보 시각·소스 명기("현재값 주장" 금지), fetch 실패/키 부재 시 해당 지역 `미연동` 폴백.
